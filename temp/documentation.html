GA4 Optimizer Documentation

GA4 Optimizer aims to solve common analysis needs when using Googele Analytics™ 4 interface. This guide explains what each feature does and how to use it. Please note that as Google Analytics 4™ interface changes some of these features may stop working.

Quick Access to Settings: Most features can be toggled on/off from the popup. Features with more detailed configurations (like "Click to Copy" actions or "Data Range Presets" visibility) have a settings option in the popup. Clicking this will open a dedicated settings page.

Features Explained

1. Click to Copy Cell

Applies to: Explorations & Standard Reports
Quickly copy values from table cells, headers, or dimension labels to your clipboard. Detailed click behaviors (single, double, triple) can be configured by clicking the Settings button in the popup.

▶
  • Single Click (if enabled):
    • Copies the text of the specific line you clicked within a cell.
    • In Explorations, if a cell contains multiple lines, the feedback message will show Line Clicked Copied!.
  • Double Click (Explorations Only, if enabled):
    • Copies the entire text content of the clicked cell or label, automatically adding spaces where lines wrap.
    • NOTE: This is not perfect. You will still have to remove spaces as you see fit.
    • Feedback message shows Copied! ⚠️Check for extra spaces.
  • Triple Click (Exploration Data Cells Only, if enabled):
    • Copies the combined text content from the hover tooltip (including Percentage of Column Total, Percentage of All Users, Segment Comparison (Δ%Shr), etc., if enabled).
    • Feedback message shows Tooltip Copied!.
Note: Copying complex multi-line text from Explorations might occasionally miss or add a space. Manual selection is always an option.

2. Sticky Report Header

Applies to: Standard Reports Only
Keeps the header row(s) of standard Google Analytics 4™ report tables visible at the top of the screen as you scroll down long tables.

  • When scrolling down a long table in a Standard Report it's hard to know which metric you're looking at. This feature keeps the header row fixed at the top.
▶

3. Highlight Sampling

Applies to: Explorations & Standard Reports
Makes the data quality/sampling icon more noticeable when your report data isn't 100% complete.

  • If the icon indicates data is sampled or incomplete (e.g., a yellow warning ⚠️), a subtle pulse animation highlights it.
▶

4. Percentage Change Highlighter

Applies to: Standard Reports & Explorations (when date comparison is active)
Visually highlights cells containing percentage changes with a clean, modern design based on user-defined ranges and colors. This helps quickly spot significant positive or negative changes in metrics when comparing date ranges. It can highlight based on the raw % change or the "Δ% Share Change" (if that feature is also active and values are present). Configure highlight ranges and colors by clicking the Settings button in the popup.

▶
  • When a Standard Report or Exploration has a date comparison active, the "% change" values will be scanned.
  • Cells containing percentage values (either raw % change or Δ% Share Change, depending on your setting) will be highlighted based on your defined ranges.
  • Default ranges are:
    • Strong Positive: 20% and above (Green)
    • Subtle Positive: 10% to 19.99% (Lighter Green)
    • Subtle Negative: -19.99% to -10% (Light Red)
    • Strong Negative: -20% and below (Red)
  • You can customize these ranges and colors in the settings to match your analysis needs.

5. Δ% Share Change (Reports)

Applies to: Standard Reports Only (when date comparison is active)
Calculates and displays the percentage change of the "percentage of total" values (share) directly in the "% change" row of Standard Reports. This provides a more insightful comparison of how a metric's share of the total has changed over time.

  • When a Standard Report has a date comparison active, this feature looks at the two date period rows and their respective "% of total" values (e.g., Metric A: 100 (10%), Metric A: 120 (12%)).
  • It then calculates the percentage change between these share percentages (e.g., a share change from 10% to 12% is a +20% change in the share).
  • This "Δ% Share Change" is appended to the existing raw value % change in the summary row. For example: -16.4% (Δ%Shr +20.00%).
Note: This feature relies on the presence of the "percent-of-total" spans (e.g., (XX.XX%)) generated by GA4 in the data rows. If these are missing or GA4 changes their structure, this feature may not work. The "Percentage Change Highlighter" can be configured to use these "Δ%Shr" values for its highlighting.

6. Percentage of Column Total

Applies to: Explorations Only
When viewing tables within Explorations, this feature automatically calculates and displays the percentage contribution of each row's value relative to the column's total.

  • The percentage value appears when you hover over a metric cell.
  • If the value is short enough, it may also appear inline, like 1,234 (12.34%).
▶
Note: If the "Total" row is scrolled out of view, percentages might temporarily show "Scroll up to capture Column's Total" or be based on the last known total.

7. Detailed Results Shortcut (Beta)

Applies to: Explorations Only
Manually trigger "More detailed results" (less sampling) in Explorations using a keyboard shortcut for better accuracy.

Requirement: This feature must be enabled in the popup and you must be on an Exploration page.
Configuration: Choose between manual and automatic activation modes by clicking the Settings button in the popup.
  • Keyboard Shortcut: Press Ctrl+Shift+Q (Windows/Linux) or Command+Shift+Q (Mac) while on an Exploration page.
  • When to Use: Use this when you see a sampling icon (⚠️) in your Exploration, indicating that the data is sampled.
  • What it Does: Attempts to switch the Exploration to use "More detailed results" which provides less sampled data.
  • Customization: You can change the keyboard shortcut by clicking the Settings button in the popup.
▶
Note: This Beta feature relies on UI automation. It might not work if GA4 loads slowly or changes significantly. In manual mode, the shortcut will have no effect if you're not on an Exploration page or if the feature is disabled. In automatic mode, the feature will attempt to trigger when relevant changes are detected on Exploration pages.

8. Percentage of All Users

Applies to: Explorations Only
Provides quick conversion rate insights by comparing segments against an "All Users" segment within Exploration tables.

Requirement: The 'All Users' segment or any segment containing naming "All users" must be included in your Exploration table setup.
  • Hover over any metric cell (except the 'All Users' column itself).
  • A tooltip shows the conversion rate: (Current Segment Cell Value) / ('All Users' Cell Value for Same Row).
▶
Visibility Requirement: If the 'All Users' column is scrolled out of view, the conversion rate might be unavailable or show 'N/A'.

9. AB Test Segment Comparison

Applies to: Explorations Only
Instantly spot statistically significant winners and losers in your A/B tests. This feature automatically detects A/B test segments and adds clear, at-a-glance visual indicators directly to your Exploration table.

▶
Requirement: Strict Segment Naming (Case-Sensitive)
  • Segments in a group must start with VAR-.
  • Control segment must start with VAR-Control (e.g., VAR-Control Original Homepage).
  • Other _VAR-_ segments are variants (e.g., _VAR-A - Blue Button_).

How it Works

  • Automatic Activation: The moment your Exploration contains correctly named segments, the feature activates automatically. No clicks required.
  • At-a-Glance Icons: Instead of requiring you to hover, the feature places minimalist arrow icons directly inside the cells of your variant segments:
    • ▲ Solid Green Arrow: Statistically significant positive uplift (≥ 95% confidence).
    • ▲ Light Green Arrow: Suggestive positive uplift (90% - 94.9% confidence).
    • ▼ Solid Red Arrow: Statistically significant negative drop (≥ 95% confidence).
    • ▼ Light Red Arrow: Suggestive negative drop (90% - 94.9% confidence).
  • Visibility Toggle: A new 'eye' icon appears in the top-right of the GA4 interface. This indicates the feature is active and allows you to temporarily hide or show the arrow icons for a cleaner view.
  • Detailed Hover Tooltip: Hovering over any cell (with or without an icon) still provides the detailed tooltip with the exact percentage differences for both raw count and Δ% Share, the statistical significance confidence level, and a **Sample Ratio Mismatch (SRM)** warning if traffic distribution is uneven.
Note: Ensure only one `VAR-Control` segment exists per metric. If the control column is scrolled out of view, comparisons may be unavailable. The statistical significance calculation is skipped for the "Totals" row.

10. Collapsible Panels

Applies to: Explorations Only
Adds a toggle button to collapse/expand "Variables" and "Tab Settings" panels in Explorations.

  • A chevron button appears on the left of the Exploration table to minimize/expand these panels.
▶
Note: Refresh the page if the button is misplaced or missing.

11. Sticky Conversion Calculator

Applies to: All Google Analytics 4™ Pages
Provides a quick-access conversion rate calculator widget.

▶
  • A calculator icon appears in the bottom-right. Click to open.
  • Calculate two CRs (CR1, CR2) by entering Conversions and Total.
  • Compares CR2 vs CR1 (Absolute Difference, Relative Uplift).
  • Panel header is draggable. Click icon or "X" to close.

12. Data Range Presets

Applies to: Standard Reports Only
Adds quick-select buttons for common date ranges (e.g., 7D, 30D, LM) and comparisons (e.g., LM MoM). Customize visible presets and their order by clicking the Settings button in the popup, or by clicking the gear icon next to the presets on the GA4 page.

▶
  • Preset buttons appear next to the date picker in Standard Reports.
  • Clicking a preset applies it and reloads the report.
  • If many presets are visible, a scrollbar appears in the preset area.
  • Comparison presets are hidden on Advertising pages.
Note: Works by changing URL parameters. Changes from the settings page should reflect after a save, or on the next report load.

13. Quick Calculated Metric

Applies to: Standard Reports & Explorations
Create temporary or saved calculated metrics (e.g., rates, ratios) directly within your reports without needing to use the GA4 Admin settings for calculated metrics. You can now save calculated rates to your personal library and choose to auto-apply them in Standard Reports, Explorations, or both. All saved metrics are managed from the settings page under Global Features.

▶

In Standard Reports, this feature adds a new, interactive column to the report table. Saved calculated rates can be set to auto-apply in Standard Reports, Explorations, or both, using the toggles in the settings page.

  • How it works: A new column appears on the far right of the table with a button to + Add calculated rate. You can choose to save your calculation for reuse and select where it auto-applies. Once a calculated rate is applied, a purple pencil icon (edit) and a bold plus (+) icon appear in the column header. The pencil icon lets you edit the current calculated rate, and the plus icon lets you add another calculated rate column instantly.
  • Configuration & Saving:
    1. Click the + Add calculated rate button inside the new column's header (if no calculated rate is present), or click the plus (+) icon in the header to add another calculated rate.
    2. A configuration modal will appear.
    3. Select the Numerator and Denominator metrics from the dropdowns. These lists are populated with the metrics currently available in your report.
    4. Choose a Display Format (Percentage or Decimal).
    5. Optionally, check "Save this calculated rate" to add it to your library. You can then choose to auto-apply it in Standard Reports, Explorations, or both using the checkboxes.
    6. Click "Apply".
  • Managing Saved Rates: All your saved calculated rates can be viewed, edited (including their name, formula, and auto-apply settings), or deleted from the Settings page (under Global Features).
  • Result: The new column will populate with the calculated rate for each row, including the "Totals" row and "% change" rows if a date comparison is active. You can re-configure the metric at any time by clicking the pencil icon in the column header, or manage your saved rates in settings. Changes to saved metrics are reflected immediately in all contexts where they are auto-applied.

In Explorations, calculated rates appear as additional lines in the hover tooltip for each metric. Multiple calculated rates can be shown for the same metric, including both on-the-fly and saved (library) metrics.

  • How it works: You designate an existing metric as the numerator, and the calculated rate appears in the main GA4 Optimizer tooltip on hover. All saved metrics set to auto-apply in Explorations will appear automatically if their numerator and denominator are present.
  • Configuration:
    1. Hover your mouse over the header of the metric you wish to use as the numerator. A small hint icon (ƒx) will appear.
    2. Right-click the metric header to open the Optimizer's context menu for that metric.
    3. Select "Create from '[Metric Name]'..." to add a new calculation (on-the-fly or saved).
    4. In the configuration modal, select the Denominator metric and display format. You can also choose to save it to your library and set auto-apply options.
    5. Click "Apply". A permanent indicator (𝑓𝑥) will now appear next to the configured metric's name in the header.
  • Result: When you hover over any data cell within that configured column, all applicable calculated rates (on-the-fly and saved) will appear as separate lines in the tooltip, each using its custom name if set. Line breaks are used for clarity.
  • Editing/Removing: Right-click the configured metric's header again. You can remove on-the-fly calculations directly from the context menu. For saved (library) metrics, use the "Edit Calculations" option to open the settings page for full management.
Note: Saved calculated metrics are available for quick access and management in the settings page. You can control where each metric auto-applies (Standard Reports, Explorations, or both). Temporary (on-the-fly) calculations are only for your current view and will be cleared when the page reloads. The tooltip always uses your custom metric name if set, and each calculation appears on its own line for clarity.

Applies to: Standard Reports & Explorations
Customize the colors of line charts and add hourly granularity to Standard Reports. Click the palette icon near charts to customize colors, or use the new "Hour" option in the time interval dropdown for detailed hourly analysis. Color preferences are configured directly via the modal and can be reset from the main settings page, accessible by clicking the Settings button in the popup.

  • When viewing a line chart, a color palette icon appears nearby. Click it to customize colors for each data series.
  • In Standard Reports, the time interval dropdown now includes an "Hour" option for hourly data granularity.
▶

15. Fix Shared Links

Applies to: All Google Analytics 4™ Pages
Fixes the common problem where shared GA4 links don't work because you're signed into the wrong Google account. When you encounter "Missing permissions" errors, this feature automatically adds a "Try Another Account" button to seamlessly retry the link with your other signed-in Google accounts.

  • Automatic Detection: When GA4 displays a "Missing permissions" dialog, the extension automatically detects it and adds a prominent "Try Another Account" button.
  • Improved Error Message: Updates GA4's confusing "You do not have access" message to a clearer version that explains it's about your default account and provides both account-switching and admin contact options.
  • Smart Retry Logic: The button systematically tries your other Google accounts (authuser=1, authuser=2, etc.) until it finds one with access or exhausts all possibilities.
  • Universal Coverage: Works on all GA4 page types including Standard Reports, Explorations, Admin pages, and any GA4 property URLs.
When This Helps: Perfect for when colleagues share GA4 links via email, Slack, or Teams, but you're signed into a personal Google account instead of your work account, or when you have multiple work accounts with different property access levels.
How It Works: The extension captures the original URL when you first navigate to a GA4 property, then if a permissions error occurs, it modifies the URL with different authuser parameters to try your other signed-in accounts. The feature automatically stops trying after checking all reasonable account indices (0-6) and displays "No Access Found" if none of your accounts have access.

16. Create Exit Pages Report

Applies to: All Google Analytics 4™ Pages (via Popup)
Instantly generates a new Standard Report template pre-filled with Exits, Views, and Bounce Rate metrics, using Page Path as the primary dimension. This feature provides a crucial workaround, as the "Exits" metric is not available in the standard GA4 Report Builder UI.

  • How it works: When you are on any GA4 property page, open the extension popup, navigate to the "Reports" tab, and click the "Create Exit Pages Report" button.
  • Your current tab will be redirected to the GA4 "Create a new report" screen, with the necessary (and normally unavailable) metrics and dimensions already selected for you.
  • You can then save this report to your library for future use. The Exit Rate column will automatically appear thanks to the calculated metric.
Note: This feature requires you to have at least Editor Role Access and to be on a page within a GA4 property (i.e., the URL must contain a property ID like "/p123456789/"). The button in the popup will be disabled if you are not on a valid GA4 page.
