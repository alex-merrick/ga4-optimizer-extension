GA4 Optimizer Documentation

GA4 Optimizer aims to solve common analysis needs when using Googel Analytics 4 interface. This guide explains what each feature does and how to use it. Please note that as GA4 interface changes some of these features may stop working.

Features Explained

1. Click to Copy Cell

Applies to: Explorations & Standard Reports
Quickly copy values from table cells, headers, or dimension labels to your clipboard.

  • Single Click:
    • Copies the text of the specific line you clicked within a cell.
    • In Explorations, if a cell contains multiple lines (like a long name or URL), the feedback message will show Partially Copied! and will only copy the direct line in the cell clicked.
    • Works on data cells, row/column headers, and dimension labels.
  • Double Click (Explorations Only):
    • Copies the entire text content of the clicked cell or label, automatically adding spaces where lines wrap.
    • NOTE: This is not perfect. You will still have to remove spaces as you see fit. Due to the way Google shows the data this is the only way to copy the full cell.
    • Feedback message shows Copied! ⚠️Check for extra spaces.
    • Note: Double-click is ignored on the first row, we're able to capture full cell value without this workaround(use Single-click there).
  • Triple Click (Exploration Data Cells Only):
    • Copies the combined text content from the hover tooltip that was created by this extention (including features like Percentage of Column Total, Percentage of All Users, and Segment Variation Comparison data, if enabled and available).
    • Feedback message shows Tooltip Copied!.
Note: This feature needs permission to modify your clipboard (to paste the copied text). Copying complex multi-line text from Explorations might occasionally miss or add a space due to how GA4 structures the text; manual selection is always an option.

2. Sticky Report Header

Applies to: Standard Reports Only
Keeps the header row(s) of standard GA4 report tables visible at the top of the screen as you scroll down long tables.

  • When scrolling down a table in a Standard Report (e.g., Pages and Screens, Acquisition reports), the header row will remain fixed at the top.
  • This helps you keep track of which column represents which metric without scrolling back up.
Known Incompatibility: This feature may not work correctly or may cause layout issues when the "Comparisons" feature (adding multiple date ranges or segments for side-by-side comparison) is active in Standard Reports. The complex table structure introduced by comparisons can interfere with how the sticky header is calculated and applied. It's recommended to disable this feature or use it with caution if Comparisons are enabled.

3. Highlight Sampling

Applies to: Explorations & Standard Reports
Makes the data quality/sampling icon more noticeable when your report data isn't 100% complete.

  • If the icon indicates data is sampled or incomplete (e.g., shows a yellow warning or red error icon), a subtle pulse animation highlights the icon. ⚠️
  • This helps draw your attention to the fact that the report data may not represent all your data.

4. Percentage of Column Total

Applies to: Explorations Only
When viewing tables within GA4 Explorations, this feature automatically calculates and displays the percentage contribution of each row's value relative to the column's total.

  • The percentage value appears when you hover over a metric cell.
  • If the value is short enough, it may also appear inline in parentheses, like 1,234 (12.34%).
  • This helps you quickly understand the relative importance of different dimension values (e.g., traffic sources, pages) for a given metric.
  • Updates automatically as you scroll through tables.
Note: This calculation appears in the combined tooltip on hover (with a blue background). Due to how GA4 Exploration tables load data (virtualization), if the "Total" row is scrolled out of view, percentages might temporarily be unavailable or based on the last known total.

5. Auto Detailed Results (Beta)

Applies to: Explorations Only

GA4 Explorations often load initially with sampled data for large data sets. This feature automatically attempts to switch the report to use less (or no) sampling ("More detailed results" setting) for better accuracy when you load an Exploration report.

  • When you open an Exploration, the extension checks the sampling status indicator.
  • If it doesn't show "100% of available data", the extension automatically selects the "More detailed results" option for you.
Note: This feature is marked as Beta because it relies on automatically selecting options in the GA4 interface. It might occasionally not work if the GA4 interface loads very slowly or changes significantly. Please allow a moment for it to complete after loading an Exploration.

6. Percentage of All Users

Applies to: Explorations Only
This feature provides quick conversion rate insights directly within Exploration tables by comparing segments against All User segments.

Requirement: For this feature to work, the 'All Users' segment must be included as one of the segments in your Exploration table setup. The conversion rate is calculated relative to this segment and the segment that you hover over.
  • Hover your mouse cursor over any metric cell (except the 'All Users' column itself).
  • A tooltip will appear showing the conversion rate calculated as: (Value in Hovered Cell) / (Value in 'All Users' Cell for the Same Row).
  • This helps you quickly compare the performance of different segments (e.g., 'Mobile Traffic', 'Converters') or dimension values against the Overall User base for that specific row item.
Note: This calculation appears in the combined tooltip on hover (with a green background).
Visibility Requirement: Due to how GA4 Exploration tables load and display data (DOM virtualization), if the 'All Users' segment column (specifically the data cell for the hovered row and metric) is scrolled too far out of view, the extension may not be able to access its value. In such cases, the conversion rate might temporarily be unavailable or show 'N/A'. Ensure the 'All Users' column is reasonably visible for consistent calculations.

7. Segment Variation Comparison

Applies to: Explorations Only
Compares performance metrics between different test variations within the same group, relative to a designated control segment. Ideal for A/B/n testing analysis directly in Explorations. This feature is enabled by default.

Requirement: Strict Segment Naming Convention (Case-Sensitive)
This feature relies entirely on how you name your segments in the Exploration setup:
  • All segments in a comparison group must start with the prefix VAR- (all uppercase).
  • The baseline/control segment must start exactly with VAR-Control (all uppercase, no hyphen or space after). Example: VAR-Control Original Homepage.
  • All other segments starting with VAR- will be treated as variants. Examples: VAR-Variant A - Blue Button, VAR-Test 2 - New Headline.
Failure to follow this convention (including incorrect capitalization) will prevent the GA4 Optimizer from identifying and comparing segments correctly.
  • Hover your mouse cursor over any metric cell belonging to a Variant Segment (one starting with VAR- but not VAR-Control).
  • A tooltip will appear showing up to two comparison lines:
    1. The percentage difference in the metric's count compared to the Control Segment count found in the same row (matching other dimensions). Calculation: (Variant Count - Control Count) / Control Count * 100%.
    2. The percentage difference in the metric's conversion rate (calculated as % of Column Total) compared to the Control Segment's conversion rate. Calculation: (Variant CR - Control CR) / Control CR * 100%.
  • Example tooltip display (colors added for clarity):
    15.20% of Column Total
    -7.50% vs Control Count
    +2.10% CR vs Control
  • Hovering over a Control Segment cell will not show these comparison lines.
Note: Ensure only one segment starting with VAR-Control exists for each unique combination of other dimensions in your table columns. The feature compares against the first matching control segment it finds. These calculations appear in the combined tooltip on hover (Count difference with a yellow background, CR difference with a red background).
Visibility Requirement: Similar to other table-based features, if the 'VAR-Control' segment column (specifically the data cell for the hovered row and metric) is scrolled too far out of view, the extension may not be able to access its value for comparison.

8. Enable GA4 Debugging

Applies to: Any Website
Enables GA4's DebugView for any website you visit while the toggle is active. This allows you to see raw event data flowing into your GA4 property in near real-time within the GA4 interface (Admin > DebugView).

  • When enabled, this feature modifies the network request for the main Google Analytics script (gtag/js) on websites you visit.
  • It adds the dbg=1 parameter to the script URL and sets the gtm_debug=LOG=x cookie in the request headers.
  • This combination signals to both GA4 and Google Tag Manager to enter debug mode, ensuring events are sent to DebugView.
  • Important: You must reload the website you want to debug *after* enabling the toggle for the changes to take effect on that site.
Side Effect: Enabling this feature also activates verbose logging from Google Tag Manager in your browser's developer console. This is necessary for DebugView to work correctly on some websites but can make the console noisy.
Note: This feature requires the declarativeNetRequest permission to modify network requests. It is designed to only modify requests to www.googletagmanager.com/gtag/js.

9. Collapsible Panels

Applies to: Explorations Only
Adds a toggle button to collapse or expand the "Variables" and "Tab Settings" panels within GA4 Explorations.

  • A small button with a chevron icon will appear on the left side of the Exploration table that allows you to minimize and expand Settings and Variables panels for easier data viewing.
Note: If the button doesn't appear or seems misplaced, try refreshing the GA4 Exploration page. The GA4 interface structure can sometimes change, which might affect the button's positioning.

10. Sticky Conversion Calculator

Applies to: All GA4 Pages
Provides a quick-access conversion rate calculator widget on all GA4 pages.

  • A calculator icon will appear in the bottom-right corner of your GA4 window.
  • Click the icon to open the calculator panel.
  • The panel allows you to calculate two independent conversion rates (CR1 and CR2):
    • Enter the number of Conversions (e.g., transactions, sign-ups) in the "Conv." field.
    • Enter the Total (e.g., sessions, users) in the "Total" field.
    • The conversion rate percentage will be displayed automatically.
  • The calculator also shows the comparison between CR2 and CR1:
    • Abs. Diff: The absolute difference in percentage points (CR2 - CR1).
    • Rel. Uplift: The relative percentage uplift of CR2 compared to CR1 ((CR2 - CR1) / CR1 * 100%).
  • Click the "Clear All" button to reset all input fields and results.
  • The calculator panel's header (displaying "Quick CR Calculator") is draggable, allowing you to move the panel around the screen if it obstructs your view.
  • Click the "X" button in the panel's header or the main calculator icon again to close the panel.
Note: This feature is designed for quick, on-the-fly calculations. The panel's position is remembered for the session after being dragged. Clicking "Clear All" does not reset the panel's position.

Troubleshooting

Ensure you are on the correct page type (Exploration or Standard Report) for the specific feature. Try refreshing the GA4 page using the button in the popup or your browser's refresh button. Double-check that the feature is toggled on in the popup.

The structure of the GA4 interface might have changed, breaking the feature. Check for extension updates or report the issue at optimizerga@gmail.com. For "Percentage of Column Total", if the "Total" row is scrolled out of view, calculations might be temporarily unavailable. For "Percentage of All Users" or "Segment Variation Comparison", if the 'All Users' or 'VAR-Control' columns respectively are scrolled out of view, the comparison data may not be available.

Make sure the 'All Users' segment is included in your Exploration columns, has a non-zero value for the relevant row, and is reasonably visible on screen. Also, ensure the feature is enabled in the popup.

  • Double-check the segment naming convention (must start with VAR-, control must start *exactly* with VAR-Control, case-sensitive).
  • Ensure both the variant and a matching control segment (with the same metric) exist in the table and are reasonably visible for the specific row/dimension combination.
  • Verify the control segment's value is not zero or non-numeric for the count comparison.
  • For the CR comparison, ensure that both the variant and control cells have valid, non-zero column totals for the internal calculation.
  • Check the browser console for errors starting with "GA4 Optimizer (Seg Comp):".

Ensure the feature is enabled. Try single vs. double vs. triple click (triple-click only works on Exploration data cells). Copying from complex Exploration tables can sometimes be imperfect due to how GA4 structures the data; manual selection might be needed.

The GA4 report structure might have changed. Try refreshing the page. If the "Comparisons" feature is active in the report, the Sticky Header may not work correctly due to the altered table structure.

Ensure the GA4 Optimizer is enabled. The highlight only appears if GA4 indicates data is sampled (yellow warning ⚠️ or red error icon). If the icon shows a green checkmark ✅ (100% data), no highlight is applied.

Ensure the toggle is enabled in the popup. Reload the website you want to debug *after* enabling the toggle. Check that no other GA debugger extensions are active. Verify in the Network tab (DevTools) that requests for `gtag/js` include `dbg=1` in the URL and the `gtm_debug=LOG=x` cookie in the request headers. If these are present but DebugView still doesn't show events, the issue likely lies within the website's specific GTM/Consent configuration overriding the debug signals.

  • Ensure you are on an Exploration page (e.g., Free form, Funnel exploration). This feature does not apply to Standard Reports.
  • The GA4 interface might have loaded in an unexpected way or changed. Try refreshing the GA4 Exploration page.
  • Check if the feature is enabled in the GA4 Optimizer popup.
  • If the button appears but is misplaced, this could also be due to GA4 UI updates. Report the issue if it persists.
  • Ensure the feature is enabled in the GA4 Optimizer popup.
  • Try refreshing the GA4 page.
  • If the calculator icon appears but the panel doesn't open, or if calculations are not working, check the browser's developer console for errors related to "StickyCalc" and report them.
